# wiki/ 폴더 규칙

이 폴더는 AI가 생성·유지하는 위키다. 사용자는 읽기만 하고, AI가 모든 작성·수정을 담당한다.

## 페이지 유형

- `index.md` — 전체 위키 목차. 모든 페이지를 카테고리별로 정리. ingest마다 갱신.
- `log.md` — 시간순 작업 기록. append-only.
- `sources/` — 원본 요약 페이지 (raw/ 파일 1:1 대응)
- `concepts/` — 개념 페이지 (Attention, Quantization, RL 등)
- `entities/` — 인물, 조직, 모델, 도구 등 엔티티 페이지
- `syntheses/` — 비교, 분석, 종합 페이지 (질의 결과 중 보존할 것)

## 페이지 작성 규칙

1. 모든 페이지에 YAML frontmatter 포함:
   ```yaml
   ---
   title: 페이지 제목
   created: YYYY-MM-DD
   updated: YYYY-MM-DD
   tags: [태그1, 태그2]
   sources: [원본 파일 경로 목록]
   ---
   ```
2. 위키링크 `[[페이지명]]` 적극 사용하여 페이지 간 연결
3. 태그 `#개념`, `#논문`, `#모델` 등으로 분류
4. 핵심(Kernel)을 먼저 서술하고, 세부사항은 그 아래에
5. 새 정보가 기존 내용과 모순되면 명시적으로 표기: `> [!warning] 모순 발견`

## 워크플로우

### Ingest (원본 → 위키)
1. raw/ 파일을 읽는다
2. 사용자와 핵심 내용을 논의한다
3. `sources/`에 요약 페이지 생성
4. 관련 `concepts/`, `entities/` 페이지 생성 또는 갱신
5. `index.md` 갱신
6. `log.md`에 기록 추가

### Query (질의 → 답변)
1. `index.md`에서 관련 페이지 탐색
2. 해당 페이지들을 읽고 종합 답변
3. 가치 있는 답변은 `syntheses/`에 저장

### Lint (위키 건강 점검)
1. 페이지 간 모순 확인
2. 고아 페이지(인바운드 링크 없음) 탐지
3. 언급되었으나 페이지가 없는 개념 식별
4. 누락된 교차 참조 보완
5. 결과를 `log.md`에 기록
