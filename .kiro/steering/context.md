## 위키 운영 규칙 (LLM Wiki 패턴)

이 볼트는 Karpathy의 LLM Wiki 패턴을 따른다. 3개 레이어로 구성:

### 아키텍처
- `01_raw/` — 불변 원본. AI는 읽기만 한다. 절대 수정 금지.
- `02_wiki/` — AI가 생성·유지하는 위키. 사용자는 읽기, AI가 쓰기.
- `03_output/` — 위키에서 파생된 최종 결과물 (블로그, 포트폴리오, 발표자료).
- `00_graphify-out/` — 그래프 데이터 (graph.json, graph.html, GRAPH_REPORT.md).

### 핵심 워크플로우

1. **Ingest** (원본 → 위키): 01_raw/에 새 파일이 들어오면 → 읽고 → 사용자와 핵심 논의 → 02_wiki/sources/에 요약 → 관련 concepts/, entities/ 갱신 → index.md 갱신 → log.md 기록
2. **Query** (질의 → 답변): index.md에서 관련 페이지 탐색 → 그래프 기반 탐색 병행 → 종합 답변 → 가치 있는 답변은 02_wiki/syntheses/에 저장
3. **Lint** (건강 점검): 모순 확인, 고아 페이지 탐지, 누락 개념 식별, 교차 참조 보완 → log.md에 기록
4. **Output** (위키 → 결과물): 02_wiki/ 기반으로 블로그/포트폴리오/발표자료 생성 → 03_output/에 저장

### 필수 규칙
- 모든 wiki 페이지에 YAML frontmatter 포함 (title, created, updated, tags, sources)
- 위키링크 `[[페이지명]]`으로 페이지 간 연결 유지
- 새 정보가 기존 내용과 모순되면 `> [!warning] 모순 발견`으로 표기
- index.md는 ingest마다 반드시 갱신
- log.md는 append-only, 형식: `## [YYYY-MM-DD] 작업유형 | 제목`

### 인과 관계 표기 컨벤션

위키링크는 "연결"만 표현한다. 지식 간의 방향성 있는 인과 관계는 아래 callout으로 명시한다.

#### 표기 형식
```markdown
> [!causal] 인과 관계
> [[원인 노드]] →(관계 동사)→ [[결과 노드]]
> 신뢰도: 높음|중간|낮음 | 출처: [[소스 페이지]]
```

#### 허용 관계 유형 (Taxonomy)
- `→(가능하게 함)→` : A가 B를 가능하게 함
- `→(성능 향상)→` : A가 B의 성능을 향상시킴
- `→(성능 저하)→` : A가 B의 성능을 저하시킴
- `→(기반이 됨)→` : A가 B의 이론적/기술적 기반
- `→(발전시킴)→` : A가 B를 발전·확장시킴
- `→(대체함)→` : A가 B를 대체함
- `→(포함함)→` : A가 B를 포함함
- `→(적용됨)→` : A가 B에 적용됨
- 필요 시 새 관계 유형 추가 가능

#### 사용 예시
```markdown
> [!causal] 인과 관계
> [[메타러닝]] →(가능하게 함)→ [[DiscoRL]]의 RL 규칙 자동 발견
> 신뢰도: 높음 | 출처: [[discovering-sota-rl-algorithms]]
```

#### Lint 시 인과 관계 점검 항목
- 순환 참조 탐지: A→B이고 B→A인 경우 `> [!warning] 순환 인과 발견`으로 표기
- 상충 관계 탐지: 동일 노드 쌍에 대해 모순되는 관계가 존재하면 격리 후 경고
- 고아 인과 탐지: 인과 관계에 언급되었으나 위키 페이지가 없는 노드 식별
