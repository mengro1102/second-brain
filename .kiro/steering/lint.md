# /lint — 위키 건강 점검 및 정리

02_wiki/ 전체를 스캔하여 깨진 링크, 모순, 누락, 오래된 정보를 찾아 수정·업데이트하는 워크플로우.

## 트리거

사용자가 "/lint", "린트해줘", "위키 정리해줘"라고 요청할 때 실행.

## 프로세스

### 1단계: 전체 스캔

1. `02_wiki/index.md`를 읽어 등록된 모든 페이지 목록을 확인한다
2. `02_wiki/` 하위 폴더의 실제 파일을 탐색한다
3. index에 등록되었으나 파일이 없는 항목, 파일은 있으나 index에 없는 항목을 식별한다

### 2단계: 링크 무결성 점검

- 깨진 링크, 고아 페이지, 누락 개념 식별

### 3단계: 내용 무결성 점검

- 페이지 간 모순 확인
- 인과 관계 순환/상충 탐지
- frontmatter 필수 필드 누락 확인

### 4단계: 수정 실행

- 자동 수정 가능: index 누락 추가, frontmatter 보완, 고아 페이지 링크 추가
- 사용자 판단 필요: 모순 해소, 인과 관계 상충

### 5단계: 기록

`02_wiki/log.md`에 점검 결과 기록.

## 규칙

- 02_wiki/ 페이지만 대상 (01_raw/는 절대 수정하지 않음)
- 자동 수정 시에도 기존 내용을 삭제하지 않고 보완한다
- 모든 수정 사항을 log.md에 기록한다
