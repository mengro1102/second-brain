# raw/ 폴더 규칙

이 폴더는 불변 원본 저장소다. AI는 이 폴더의 파일을 절대 수정하지 않는다.

## 구조

- `papers/` — 논문 PDF, 마크다운 변환본
- `articles/` — 웹 기사, 블로그 클리핑 (Obsidian Web Clipper 등)
- `code/` — 코드 스니펫, 실험 코드, 노트북
- `notes/` — 수기 메모, 아이디어, 브레인스토밍
- `transcripts/` — 유튜브 자막, 강의 노트, 미팅 기록
- `assets/` — 이미지, 다이어그램 등 첨부파일

## 규칙

1. 원본은 불변이다. 읽기 전용.
2. AI가 ingest할 때 이 폴더에서 읽고, wiki/에 정리한다.
3. 가능하면 YAML frontmatter를 포함한다:
   ```yaml
   ---
   title: 제목
   source: URL 또는 출처
   date: YYYY-MM-DD
   tags: [태그1, 태그2]
   type: paper | article | code | note | transcript
   ---
   ```

## 네이밍 컨벤션

유형별로 파일명 형식이 다르다:

| 유형 | 형식 | 예시 |
|------|------|------|
| 논문 (papers/) | `{제목-슬러그}/{제목-슬러그}.md` | `discovering-sota-rl-algorithms/discovering-sota-rl-algorithms.md` |
| 기사 (articles/) | `{YYYY-MM-DD}_{제목-슬러그}.md` | `2026-04-14_llm-wiki-knowledge-base.md` |
| 유튜브 (transcripts/) | `{YYYY-MM-DD}_{제목-슬러그}.md` | `2026-04-14_llm-wiki-truth-pkm-fails.md` |
| 코드 (code/) | `{YYYY-MM-DD}_{제목-슬러그}.md` | `2026-04-14_discorl-experiment.md` |
| 메모 (notes/) | `{YYYY-MM-DD}_{제목-슬러그}.md` | `2026-04-14_rl-idea-brainstorm.md` |

### 슬러그 규칙
- 영어 소문자 + 하이픈 (kebab-case)
- 한국어 제목은 핵심 키워드를 영어로 변환하여 슬러그 생성
- 최대 80자

### PDF 리네이밍 (marker 변환 후)
PDF를 `marker_single`로 변환하면 원본 파일명 기반 폴더가 생성된다.
변환 후 제목이 식별되면 다음을 수행한다:
1. 변환된 마크다운에서 제목(# 헤딩 또는 title 메타데이터)을 추출
2. 제목을 슬러그로 변환
3. PDF 파일, 변환 폴더, 마크다운 파일을 모두 슬러그 기반으로 리네이밍
   - `s41586-025-09761-x.pdf` → `discovering-sota-rl-algorithms.pdf`
   - `s41586-025-09761-x/` → `discovering-sota-rl-algorithms/`
   - `s41586-025-09761-x.md` → `discovering-sota-rl-algorithms.md`

### 숫자 인덱싱 ID는 사용하지 않는다
- 문서 간 연결성은 위키링크(`[[]]`)와 인과 관계 callout이 담당
- 숫자 ID는 의미가 모호하고 관리 마찰만 증가시킴
- 고유성은 날짜+제목 슬러그 조합으로 자연스럽게 보장
