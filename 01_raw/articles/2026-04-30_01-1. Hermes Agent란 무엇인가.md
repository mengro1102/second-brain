## 원문

### 개요

Hermes Agent는 NousResearch가 만든 오픈소스 AI 에이전트입니다. 2026년 2월 26일에 공개되었고, MIT 라이선스로 누구나 자유롭게 사용할 수 있습니다.

OpenClaw가 큰 인기를 끌면서 비슷한 개인용 AI 에이전트가 속속 등장하고 있는데, Hermes Agent도 그중 하나입니다. 터미널에서 `hermes` 를 입력하면 대화가 시작되고, 파일을 읽고 쓰고, 웹을 검색하고, 명령을 실행하고, 메시지를 보내는 등 다양한 작업을 수행합니다. 내장 도구 29개에는 터미널·파일 같은 개발 도구와 함께 브라우저 자동화, 이미지 생성, 음성 합성, 스마트 홈 제어가 포함되어 있고, 94개 내장 스킬의 카테고리도 소프트웨어 개발, MLOps부터 게임, 음악, 생산성, SNS, 스마트 홈까지 폭넓게 걸쳐 있습니다.

메모리와 스킬로 사용할수록 똑똑해지는 "성장하는 에이전트"는 OpenClaw를 비롯해 이 분야의 공통 흐름입니다. Hermes Agent도 같은 방향이지만, 복잡한 작업이 끝난 뒤 **백그라운드에서 대화를 리뷰하고 스스로 스킬을 생성** 하는 자동화 메커니즘으로 한 단계 더 나간 것이 특징입니다(5장 메모리, 7장 스킬 참조).

### 활용 사례

커뮤니티에서 공유된 실제 활용 사례를 소개합니다.

- **Mac Mini를 24시간 AI 비서로**: Mac Mini에 Hermes Agent를 설치하고, Apple 미리알림·메모·iMessage와 연동해서 "내일 오전 10시에 치과 예약 알려줘"라고 Telegram으로 보내면 끝. 크론으로 매일 아침 일정과 날씨를 브리핑합니다. (→ 13-4장)
- **팀 디스코드 봇으로 반복 질문 해결**: "스테이징 서버 상태 어때?", "이 PR 리뷰해줘" 같은 반복 질문을 디스코드 채널의 에이전트가 처리. 처음에는 매번 방법을 찾지만, 스킬이 쌓이면서 점점 빨라집니다. (→ 13-3장)
- **매일 뉴스 브리핑 자동화**: "매일 오전 9시에 AI 뉴스를 요약해서 텔레그램으로 보내줘." MCP로 검색 API를 연결하고 크론으로 등록하면, 개인 맞춤 뉴스레터가 매일 도착합니다. (→ 13-2장)
- **$5 VPS에서 돌리는 완전 무료 AI 비서**: 로컬 LLM(Hermes 3, 8B)을 연결하면 API 비용 없이 에이전트를 운영할 수 있습니다. 다만 로컬 모델만으로는 복잡한 지시나 한국어 처리에 한계가 뚜렷하므로, 클라우드 API와 병행하는 혼합 전략이 현실적입니다. (→ 2-3장)

이런 활용이 가능한 이유는 Hermes Agent가 **도구 + 메모리 + 스킬 + 메시징 + 크론** 을 하나로 통합하고 있기 때문입니다. 각 기능은 이 책의 해당 장에서 자세히 다룹니다.

### 핵심 기능

#### 대화형 CLI

터미널에서 `hermes` 를 실행하면 AI와 대화가 시작되고, 웹 검색, 파일 작업, 명령 실행 등을 자연어로 지시할 수 있습니다.

#### 유연한 모델 연동

OpenAI, Anthropic, Ollama(로컬 모델), OpenRouter(200개 이상 모델) 등 OpenAI 호환 API면 무엇이든 됩니다. 이 책에서는 로컬 LLM(Ollama)과 클라우드 API(Claude 등)를 함께 활용하는 방법을 다룹니다.

#### 멀티채널 메시징

Telegram, Discord, Slack, WhatsApp, Signal에서 봇으로 동작합니다. 터미널 앞에 앉아 있지 않아도 스마트폰에서 지시를 내릴 수 있습니다.

#### 예약 실행과 자동화

크론 스케줄링으로 무인 작업이 가능합니다. "매일 아침 8시에 뉴스를 요약해서 Telegram으로 보내줘" 같은 자연어 한 문장이면 됩니다.

#### 메모리와 스킬 학습

대화 내용을 기억하고, 사용자의 선호를 학습하고, 복잡한 작업의 절차를 스킬로 저장합니다. 같은 종류의 작업을 반복하면 점점 빨라집니다.

#### 52개 내장 도구와 MCP 확장

웹 검색, 파일 읽기/쓰기, 터미널 명령, 브라우저 자동화, 이미지 분석, 코드 실행, 서브에이전트 위임 등 다양한 도구가 내장되어 있습니다. 기본 도구 세트만 29개이고, Home Assistant·크론·메시징 등의 확장 도구까지 합치면 52개에 이릅니다. MCP(Model Context Protocol)로 외부 도구를 추가할 수도 있습니다.

### 기술 스택

- **아키텍처**: ReAct(Reasoning and Acting) 루프. 관찰 → 추론 → 행동을 반복합니다
- **기본 모델**: Hermes-3 (Llama 3.1 기반). NousResearch가 Atropos라는 강화학습 프레임워크로 도구 호출 능력을 학습시켰습니다
- **코드**: Python 91.4%. 나머지는 Shell, JavaScript 등
- **라이선스**: MIT
- **최신 버전**: v0.7.0 (2026년 4월 3일)
- **GitHub**: 약 8,900 스타 (2026년 3월 기준)
