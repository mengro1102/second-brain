### 왜 Hermes Agent를 선택하는가

AI 에이전트 프레임워크는 이미 많습니다. 선택지가 넘쳐나는 시점에서, Hermes Agent가 가진 매력을 정리합니다. 다른 에이전트와의 구체적인 비교는 1-3절에서 다룹니다.

### 학습하는 에이전트

Hermes Agent는 세 가지 메커니즘으로 **세션을 넘어 학습합니다**.

1. **메모리**: 환경 정보, 규칙, 교훈을 기억합니다
2. **사용자 프로필**: 사용자의 선호와 작업 스타일을 파악합니다
3. **스킬 자동 생성**: 복잡한 작업을 수행한 뒤, 백그라운드 리뷰 에이전트가 스킬 저장 여부를 검토합니다

메모리와 사용자 프로필이 "누구의 어떤 환경인지"를 기억하고, 자동 생성된 스킬이 "어떻게 처리하는지"를 축적합니다. 이 조합이 "함께 성장하는 에이전트"라는 슬로건의 의미입니다.

### 완전한 데이터 주권

로컬 LLM(Ollama, vLLM, llama.cpp)과 연동하면 데이터가 한 바이트도 외부로 나가지 않습니다. **$5짜리 VPS에서도, GPU 클러스터에서도, 서버리스 환경에서도** 구동됩니다.

`config.yaml` 에서 모델과 엔드포인트를 지정하면 완전한 로컬 실행이 됩니다. 구체적인 설정 방법은 2-3절을 참고하세요.

### 모델이 아니라 하네스에 가치가 쌓입니다

AI 에이전트는 **모델(LLM) + 하네스(harness)** 로 구성됩니다. 모델이 "무엇을 할지" 생각하는 뇌라면, 하네스는 그 뇌를 실제로 유용하게 작동시키는 나머지 전부입니다.

- 도구 사용(터미널, 브라우저, API 연결)
- 메모리와 컨텍스트 관리(장기 기억, 세션 간 상태 유지)
- 실행 루프(계획 → 실행 → 검증 → 수정)
- 안전장치(권한 제어, 승인, 오류 복구)
- 스킬 자동 생성과 자기 수정

같은 모델이라도 하네스에 따라 에이전트의 성능이 크게 달라집니다. Hermes Agent는 이 하네스가 핵심 경쟁력이며, 모델은 언제든 교체할 수 있습니다. 오늘은 Ollama의 Llama 3.1을 쓰다가, 내일은 Claude Sonnet으로, 다음 주에는 Gemma 4로 바꿔도 메모리, 스킬, 워크플로가 그대로 유지됩니다.

### 자율 백그라운드 작업

크론 스케줄링과 메시징 게이트웨이를 통해 **무인 자율 작업** 이 가능합니다.

- 매일 아침 뉴스를 요약해서 Telegram으로 전송
- 매주 코드 저장소를 분석하여 리포트 생성
- 야간에 백업 작업 수행

반복 작업에서 스킬이 자동 생성되므로, 시간이 지남에 따라 더 정확하게 수행합니다.

### 에이전트가 필요한 작업인가

에이전트는 만능이 아닙니다. 모든 작업에 에이전트를 쓰면 오히려 비효율적입니다.

- **문서 검색·요약·Q&A**: RAG(Retrieval Augmented Generation)가 더 적합합니다. 문서 전체를 컨텍스트에 넣는 대신 관련 부분만 검색해서 전달하므로 토큰 낭비가 적습니다.
- **단순 대화**: `ollama run` 이나 API를 직접 호출하는 것이 빠르고 효율적입니다.

에이전트의 진짜 가치는 **도구를 동적으로 호출하고, 여러 단계를 계획하며, 외부 액션(파일 쓰기, API 호출, 브라우저 제어 등)을 수행해야 할 때** 발휘됩니다. 부록 "로컬 모델 비교 테스트"에서 이 트레이드오프를 실측 데이터와 함께 자세히 다룹니다.

### 비용

로컬 LLM을 사용하면 하드웨어 전기 요금 외에 추가 비용이 없습니다. 클라우드 API를 함께 쓰더라도, 스마트 모델 라우팅으로 간단한 질의는 작은 모델로, 복잡한 작업은 큰 모델로 자동 배분하여 비용을 최적화할 수 있습니다.

### 설계 수준의 보안

AI 에이전트는 사용자의 자격 증명으로 코드를 실행하고, 파일을 읽고, 외부 서비스에 접근합니다. 편리하지만 그만큼 위험합니다. 2026년 2월 MS 보안 블로그는 OpenClaw를 사례로 들어, AI 에이전트의 네 가지 핵심 위험을 경고했습니다.

| MS가 지적한 위험 | Hermes Agent의 대응 |
| --- | --- |
| 위험한 명령의 무분별한 실행 | `rm -rf`, `curl \\| sh` 등 20개 이상의 위험 패턴을 감지하고 실행 전 사용자 승인을 요구합니다 |
| 호스트 권한을 그대로 상속 | Docker 등 컨테이너 백엔드로 호스트와 격리하고, MCP 서버에는 화이트리스트 환경변수만 전달합니다 |
| 악성 스킬을 통한 공급망 공격 | 스킬을 에이전트가 직접 생성하고, 백그라운드 리뷰 에이전트가 자동 검토합니다 |
| 간접 프롬프트 주입 | 컨텍스트 파일 로드 전에 인젝션 패턴과 비가시 유니코드를 스캔하여 차단합니다 |

MS는 "격리된 환경에서 실행하고, 전용 계정을 쓰고, 지속적으로 모니터링하라"고 권고했습니다. Hermes Agent는 이 권고 사항을 사용자가 직접 구성하는 대신 **5계층 보안 모델** 로 아키텍처에 내장했습니다. 각 계층의 구현 상세는 11장에서 다룹니다.

### OpenClaw에서 전환하기

이미 OpenClaw를 사용 중이라면, Hermes Agent로의 전환은 간단합니다.

```bash
hermes claw migrate              # 대화형 마이그레이션
hermes claw migrate --dry-run    # 미리보기 (실제 변경 없음)
hermes claw migrate --preset user-data  # 시크릿 제외
```

가져올 수 있는 항목:

- 페르소나 파일
- 메모리 항목
- 사용자 생성 스킬
- 명령 허용 목록
- 플랫폼 설정
- API 키 (선택적)
- TTS 에셋
- 워크스페이스 지시사항

### 스킬 공유 생태계

Hermes Agent의 스킬 시스템은 **agentskills.io 오픈 표준** 을 따릅니다. 이 표준은 Anthropic이 주도하고 Claude Code, Cursor, Gemini CLI 등 16개 이상의 도구가 채택한 업계 공통 규격입니다. 스킬은 사람이 읽을 수 있는 마크다운 형식이며, 감사(audit) 가능하고, 에이전트 간에 공유할 수 있습니다. Skills Hub를 통해 커뮤니티가 만든 스킬을 설치하고, 자신이 만든 스킬을 공유할 수 있습니다.
